# Спец для разворота статей портала «Архитектура сознания»

Задача: развернуть статью в 4-7 раз (полноценный лонгрид ~2500-3500 слов), НЕ водой, а глубиной: механизмы, примеры, эксперименты, честные оговорки, практические шаги. Каждый абзац несёт факт или мысль, за которую сохранят.

## Формат вывода
Пишешь Python-выражение из хелперов, склеенных через `+`. Записываешь его в файл `genwork/art-<slug>.py` РОВНО так:
```
BODY = (
  P('...')
  + H('...')
  + P('...')
  ...
)
```
Ничего кроме `BODY = (...)`. Без ```-заборов, без комментариев, без пояснений.

## Хелперы (вызывай и склеивай через +)
- `P('текст')` — абзац. Внутри можно `<b>жирный</b>`.
- `H('Подзаголовок')` — заголовок раздела (h2). Живой, не канцелярский, интригует.
- `pullq('цитата без кавычек','Автор, источник')` — выносная цитата.
- `factbox('Заголовок','Текст врезки','Источник','КЛАСС')` — врезка-факт. КЛАСС: `''` синий (мозг/наука), `'t'` бирюзовый (тело), `'m'` золотой (деньги).
- `afig('img/ФАЙЛ.jpg','подпись под картинкой')` — картинка в середине текста.
- `artstats([('ЧИСЛО','подпись'),('ЧИСЛО','подпись'),('ЧИСЛО','подпись')], hi=0, hic='')` — три крупных числа, hi = индекс подсветки (0..2), hic = '' / 't' / 'm'.
- `pts('пункт','пункт',...)` — маркированный список. В пунктах можно `<b>...</b>`.

## Насыщенность лонгрида (обязательно)
- 7-10 разделов `H(...)`.
- 2-3 картинки `afig` (РАЗНЫЕ файлы, не повторять внутри статьи).
- 1-2 блока `artstats` с честными числами.
- 4-6 `factbox` с реальными источниками.
- 2-4 `pullq`.
- 1-2 списка `pts`.
- Много содержательных `P` между ними.

## Голос и железные запреты
- Живой грамотный русский, как знающий человек. Чередуй ритм: после длинного предложения короткое.
- НИ ОДНОГО длинного/среднего тире (— –). Только запятая, двоеточие, точка, скобки. В числах диапазон через дефис (16-28).
- Никаких ИИ-штампов: является, представляет собой, играет ключевую роль, в современном мире, стоит отметить, таким образом, широкий спектр, delve, robust, seamless и т.п.
- НИКОГДА «это не эзотерика / не магия / не мистика / не философия». Подавать позитивно через факт («механизм проверен», «за это дали Нобеля»).
- Без редких/неживых слов: всплеск, наутро, мощь, вмиг, уклад, плеск, гудение, обёртка, почуял. Простые слова.
- Факты РЕАЛЬНЫЕ, атрибуция честная (автор, работа, год). Ничего не выдумывать. Где наука ранняя или спорная, честно: «направление, а не финал», не обещать чудес.
- Кавычки «ёлочки». Конкретика: числа, имена, детали.
- Не заканчивать водянистым резюме. Финал — сильная мысль, которая остаётся.
- Минимум «ты»-тыканья, писать больше через «человек», «мы», безлично; «ты» дозированно.

## Апострофы в Python-строках
Строки в одинарных кавычках. Русские кавычки « » апострофов не создают. Если внутри нужен апостроф латиницы (Don't) — избегай, пиши по-русски. НЕ используй двойные кавычки для строк, только одинарные.
